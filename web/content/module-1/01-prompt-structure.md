## 第 1 节：核心概念

### 一个好的提示词，长什么样？

大多数人写提示词的方式是：想到什么说什么。结果模型的输出时好时坏，不知道哪里出了问题。

Anthropic 通过大量测试发现，高质量提示词通常由五个区块构成。不是所有任务都需要全部五块，但**当你的提示词效果不好时，往往是缺少了其中某几块**。

### 五要素结构

| 要素 | 作用 | 关键问题 |
|------|------|----------|
| **① 任务目标** | 告诉模型做什么 | 你想要什么输出？面向谁？什么格式？ |
| **② 动态内容** | 提供每次变化的数据 | 这次调用，哪些信息会变？ |
| **③ 步骤指令** | 规定分析或行动的顺序 | 应该按什么逻辑来思考这个问题？ |
| **④ 示例** | 注入人类判断力 | 有哪些边界情况需要用例子来说明？ |
| **⑤ 收尾要求** | 重申关键约束 / 控制输出格式 | 有什么不能遗漏的最终要求？ |

这五个要素没有固定顺序，也不必全部出现。**它们是一个检查清单，而不是填空模板**：每次写完提示词，逐一对照，找出哪一块缺失了，补上就好。

### 场景 A：产品经理分析用户反馈

```
你是一位资深产品经理，擅长从用户反馈中提炼产品需求。

<feedback>
{{本周收集到的用户反馈（10-30条）}}
</feedback>

请按以下步骤分析：
1. 逐条标注反馈类型（功能/体验/性能/其他）
2. 统计各类型数量
3. 找出被提及超过 3 次的核心问题
4. 按影响程度排序，输出优先级矩阵

输出格式：先给总览表格，再给每条问题的详细说明

<analysis>
```

---

### 场景 B：运营分析周报异动

```
你是数据分析师，负责每周分析运营指标并标记异常。

判断标准：环比变动超过 20% 为异常

<data>
{{本周运营数据 CSV}}
</data>

步骤：
1. 计算每个指标的环比变动率
2. 标记所有超过 ±20% 的指标
3. 为每个异常给出 1-2 个可能原因（标注置信度：高/中/低）
4. 给出建议的跟进动作

不确定的原因用【待确认】标注，不要猜测
```

---

### 场景 C：市场撰写用户案例

```
你是科技公司的市场文案，擅长写 B2B 客户案例。

<interview>
{{客户访谈原文}}
</interview>

<product_info>
{{产品核心卖点（3条）}}
</product_info>

示例结构：
- 痛点（1句）→ 解决方案（2句）→ 成果（量化数据）

要求：400字以内，语气真实，避免空洞形容词（如"显著"、"大幅"）
必须包含至少1个客户的原话引用
```

---

注意三个场景的共同规律：**① 角色定位在最前，② 动态数据用占位符标记，③ 步骤用数字编号，④ 格式要求放在末尾**。这四个习惯可以立刻改善大多数提示词。

## 附录：练习 1 参考答案

原提示词：「总结这篇文章的核心观点」

- **缺少动态内容**：没有提供文章内容，模型无从下手，只能瞎编
- **缺少角色定位**：是给谁总结的？学者？经理？普通读者？受众不同，深度和用词都应不同
- **缺少步骤指令**：先看整体结构，还是直接找关键句？没有指引，模型随机选择方式，结果不可预测
- **缺少输出格式**：几句话？几条要点？需要原文引用吗？格式不定，输出长短随机

这是最常见的「最简版提示词」。它能用，但质量完全依赖模型猜测，不稳定。
