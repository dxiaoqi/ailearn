## 1 · 为何先做需求卡？

Vibe Coding 中，**需求越清楚，Trae 越接近你想要的实现**。避免「好看一点」这类模糊表述；改为具体可见元素：深色主题、卡片栅格、标题/摘要/市场影响标签、外链等。

---

## 2 · 给 Trae 的整体需求（示例）

以下可复制为起点，再按你的需求卡个性化：

```
我已经有两个 Python 脚本：
- fetch_news.py：运行后从 AP News 抓取金融新闻，保存到 data/summary.json 的 raw_news 字段
- news_agent.py：运行后读取 raw_news，用 Claude AI 整理，结果保存到 items 字段

items 的格式：[{"title":"...","summary":"...","market_impact":"...","url":"..."}]

我想把这个系统做成一个网页产品，需求：

【展示】
用卡片展示 items 里的每条新闻，深色专业风格（参考金融资讯网站）
每张卡片：标题 / 摘要 / 市场影响（标签样式）/ 查看原文链接

【交互】
一个"立即刷新"按钮，点击后：
1. 触发 fetch_news.py 和 news_agent.py 的运行
2. 按钮显示 loading 状态（不能点了没反应）
3. 数据更新后自动刷新页面内容

【自动】
每天 08:00 自动运行一次

你来决定用什么技术方案，我只关心最终效果。
```

**教学点**：若 Trae 选择 **Flask**，可解释——浏览器不能直接运行本机 Python，需要**后端**接收请求、触发脚本、再把 JSON 返回前端；不必深入框架，只要能**运行与验收**。

**迭代示范**（对 Trae 说人话）：卡片太挤；刷新无反馈；移动端要单列布局等。

## 3 · 学员自主完成

1. 用自己的需求卡描述，不要照抄讲师稿。  
2. 遇错先把**现象**告诉 Trae（例如「按钮可点但列表不变」）。  
3. 按 Trae 说明启动（常见 `python app.py`），浏览器打开 `http://localhost:5000`（以实际端口为准）。  
4. 验收：卡片展示 / 刷新有 Loading / 数据确实更新。

**讲师角色**：少给代码答案；多问「你怎么向 Trae 描述现象」。

**常见卡点**：启动报错贴给 Trae；白屏可查 `/api/news` 是否返回 JSON；按钮无 Loading 可查前端点击处理。

## 5 · JSON 初识（为第二期铺垫）

**问题**：另一段程序或 Agent 要读 `market_impact` 并推断「资产类别、涨跌方向」——纯自然语言**不易稳定解析**。

### 三条规则（快讲）

1. **每个值有名字**：`"down"` 不如 `{"direction": "down"}`。  
2. **类型明确**：百分比用数字，如 `"change_pct": -1.2`，避免 `"下跌1.2%"` 混在字符串里难以计算。  
3. **结构可嵌套**：多条影响可放数组，如 `impacts: [{asset, direction}, ...]`。

### 小练习（约 3 分钟）

原文：「受美联储加息预期影响，美元走强，黄金承压」。

参考答案（字段可自拟，一致即可）：

```json
{"trigger": "美联储加息预期", "impacts": [{"asset": "USD", "direction": "up"}, {"asset": "Gold", "direction": "down"}]}
```
