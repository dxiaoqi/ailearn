## 1 · 开场：从「手动整理信息」谈起

**核心问题**：你每天花多少时间在手动整理信息上？如果有一位实习分析师代劳，你会怎么交代任务？

引导学员各用一句话描述场景（刷新闻、研报摘要、晨会纪要等）。在白板或文档中可按 **信息来源 / 输出形态** 两列归类——后续练习会反复用到这些真实场景。

**过渡**：这些工作共性是「信息进来 → 处理 → 输出结论或摘要」。本课问题：**能否交给 Agent？** Agent 与日常用的 ChatGPT 有何本质不同？

---

## 2 · 核心概念：什么是 Agent？

日常用法往往是「你问一句、模型答一句」，**你是主导，模型被动响应**。

**Agent** 更接近「能派活的实习分析师」：你交代「每天早上整理 AP News 里与金融市场相关的要闻」，它可以自己去取数、用工具、判断重要性、写出摘要，完成后再汇报。

| 普通对话式 Chat | AI Agent |
|-----------------|----------|
| 一问一答，需人在场 | 交代任务后可异步运行 |
| 通常不主动调用外部工具 | 可抓取网页、读写文件等 |
| 多轮需人持续推进 | 可在规则内多步重试直至结束 |

**类比**：实习生听懂「每天早上 8 点去 AP News 看金融相关新闻，整理 5 条摘要发群」后，可重复执行；不确定时再问你。Agent 的核心特质是：**一定自主性、会用工具、能处理部分异常**。

> **与 RPA 的对比（常见问题）**：RPA 像录好的宏，页面改版易失效；Agent 依赖语义理解与判断，更像人在读上下文后再决定下一步。

## 3 · 三种节点类型（设计工具）

设计系统时，**并非每一步都需要大模型**。区分清楚，系统才稳。

1. **Agent 节点**：需要理解语义、综合判断、难以用固定规则覆盖。  
   *例：判断新闻重要性、撰写摘要、关联市场影响。*2. **Automation 节点**：按确定规则执行，无需「理解」。  
   *例：定时触发、保存文件、格式转换、发邮件。*

3. **Human 节点**：责任大、合规敏感或强主观，必须人拍板。  
   *例：最终投资决策、风控审批、策略方向。*

**快问快答**：

- 每天早上8 点自动跑一次抓取 → **Automation**  
- 判断一条新闻对股市是否有影响 → **Agent**  
- 把摘要写入文件 → **Automation**  
- 决定是否今日调仓 → **Human**

**原则**：Agent 节点通常更贵、更慢、更易出意外。**能用 Automation 就不用 Agent；必须人做的不要绕过 Human。**

> 边界模糊时，问一句：这一步需要**语义理解或主观判断**吗？需要 → Agent；不需要 → Automation。仍模糊时，可优先 Automation 以换取稳定性。

## 4 · 练习①：AP News 整理流程图

**场景**：用 Agent **自动**整理 AP News 的金融相关新闻，每天早上产出一份摘要。

**要求**：

1. 每个步骤标注节点类型：**Agent / Automation / Human**。  
2. **至少一条失败分支**（例如：AP News 无法访问、列表为空、超时）。

**点评时可关注**：

- 是否把「保存文件」误标成 Agent；  
- 失败分支是否具体（仅写「报错」不够，需说明之后行为）；  
- 是否遗漏「数据存哪」「谁触发」等步骤。

## 5 · 练习②：News Agent 角色卡

流程图描述「系统长什么样」；**角色卡**描述「Agent 节点内部是谁、做什么」——相当于岗位说明书。本课只用自然语言，**下节课**将升级为可执行的 Prompt。

建议五字段：

| 字段 | 含义 |
|------|------|
| 目标 | 该 Agent 要达成什么业务结果 |
| 输入 | 能看到哪些材料（来源、条数、字段） |
| 处理 | 筛选、排序、提炼规则（文字描述即可） |
| 输出 | 条数上限、每条包含哪些信息 |
| 失败时 | 无新闻、源站异常、内容不足时如何表现（禁止编造） |

**示范（供讲师参考，不必原样发学员）**：

- **目标**：从当日 AP News 中提炼金融市场最需关注的信息。  
- **输入**：当日列表，最多 20 条，含标题、时间、摘要等。  
- **处理**：筛金融相关，按重要性排序，提炼标题级摘要与市场关联。  
- **输出**：最多 5 条；每条含原标题、2–3 句摘要、为何从业者需关注。  
- **失败时**：无相关新闻则明确输出「今日无重大金融新闻」，不编造。

---

## 6 · 常见问题（简要）

- **Agent 很难学吗？** 本路径侧重把需求说清楚（像给实习生写任务说明），难点往往在需求澄清而非语法。  
- **学完真能跑起来吗？** 第 4 课在终端跑通脚本，第 5 课可做成可分享的网页产品。  
- **为何选 AP News？** 免费、权威、结构相对稳定，便于教学；后续可换数据源。
