回顾 · 0–5分钟
老师开场
上节课大家画了流程图、填了角色卡。我挑了几份分享一下——

（展示2~3份学员角色卡，重点看"输出"和"失败时"两个字段）

我注意到一个规律：大部分人写输出的时候，描述了"是什么"，但没有写清楚"长什么格式"。 比如写"输出新闻摘要"——但是几条？每条包含哪些字段？纯文字还是有结构的？

这正是今天要解决的问题：怎么把角色卡里的描述，变成 AI 真的能执行的指令。
开场实验 · 5–20分钟
同样让 AI 整理新闻，随意问和结构化问，差别有多大？
实验操作说明
准备一段今日 AP News 的真实新闻段落（约200字）。

第一次提问（随意方式）：
"帮我整理一下今天的 AP 金融新闻，给我看看有什么重要的。"
把输出展示给学员看（通常：格式不固定、字数随意、有时候没有市场关联分析）。

第二次提问（四层结构 Prompt）：
使用下方完整的示例 Prompt，把同一段新闻作为输入。
把输出展示给学员看（格式固定、字段清晰、有市场影响分析、JSON 格式）。

引导学员说出差别：格式、完整性、可用性。
过渡引导
差别在哪？不是第二次问的 AI 更聪明——是我们的指令更清晰。

第一次提问，门是开着的，AI 可以随意发挥。
第二次提问，我们把门关严了：什么能进来、怎么处理、怎么出去——全都定义清楚了。

这就是今天的核心概念：Prompt 是逻辑门。
核心概念 · 20–40分钟
Prompt 四层结构——关严逻辑门
核心类比 Prompt 就像岗位说明书 + 当日任务书。写得越清楚，AI 发挥的空间越小，输出越稳定。 "让 AI 自由发挥"在金融场景里通常是坏事——我们需要的是可预期、可验证的输出。
1
角色设定
你是谁 + 你的专业边界 + 你只能做什么
金融类比：岗位职责 —— "本岗位负责…不负责…"
2
上下文引入
今天你能看到哪些信息（输入边界）——明确信息从哪来、有多少
金融类比：今日工作材料包 —— "本次分析基于以下数据…"
3
任务指令
处理规则 + 输出格式要求（几条、什么字段、JSON 还是文字）
金融类比：任务说明书 —— "请按以下格式提交报告…"
4
约束规则
禁止事项 + 异常情况处理（如果…则返回…）
金融类比：合规红线 —— "以下情况须上报合规部门…"
完整示例：News Agent Prompt
【第1层：角色设定】
你是一位专注于金融市场的新闻分析师。
你只基于我提供的新闻列表进行分析，不引用任何我没有提供的内容。
不得凭空编造任何新闻或数据。

【第2层：上下文引入】
以下是今日 AP News 的新闻列表（最多20条）：
{news_list}

【第3层：任务指令】
筛选与金融市场相关的新闻，按重要性从高到低排序。
对每条相关新闻输出以下字段：
- title：保持原标题，不改写
- summary：2~3句话，说清楚发生了什么
- market_impact：1句话，为什么金融从业者需要关注
- url：原文链接
最多输出5条，以 JSON 格式输出：{"items": [...]}
不要包含任何额外说明文字。

【第4层：约束规则】
- 娱乐/体育/生活新闻直接跳过，不要提及
- 如果今日无金融相关新闻，返回：{"items": [], "message": "今日无重大金融相关新闻"}
- 不得编造任何内容，所有摘要必须来自提供的原文
为什么要指定 JSON 格式？ 第4节课，我们需要让另一个程序读取 AI 的输出。程序读不懂自然语言，但能精确读取 JSON 的字段值。 从现在开始养成"输出 JSON"的习惯，后面不需要额外改造。

练习② · 70–85分钟
互换互检——用"刁难问题"找漏洞
互检说明
两人互换 Prompt，扮演"挑剔的程序员"，用以下问题检验对方的 Prompt：

① 如果今天 AP News 服务器挂了，新闻列表是空的，AI 会输出什么？
② 如果 AI 输出了6条新闻（超过5条上限），怎么处理？
③ 如果 AI 输出的格式是纯文字而不是 JSON，你的程序能读取吗？
④ 如果有一条娱乐新闻混进来了，AI 会把它过滤掉吗？

找到漏洞后，告诉对方怎么修——不是嘲讽，是帮助对方关严逻辑门。
老师点评 · 85–95分钟
四大常见漏洞
漏洞类型典型表现后果修复方式
角色太模糊 "你是一个 AI 助手" AI 不知道专业边界，可能胡乱引用外部信息 加专业领域 + "只基于我提供的内容"
没有输出格式 "整理成摘要给我" 每次格式不同，程序无法稳定读取 明确每条含哪些字段，指定 JSON 格式
没有数量限制 （未说明） AI 可能输出2条也可能输出15条 明确"最多输出N条"
缺失异常处理 第4层为空 无新闻时 AI 可能编造，或格式崩掉 至少加一条：无数据时返回空 JSON
重点提醒 JSON 格式是这节课最重要的产物之一。第4节课生成的代码会直接依赖这个格式。 如果学员的 Prompt 里没有指定 JSON 输出，下节课搭建项目时会遇到问题。

常见问题参考回答
  • "JSON 是什么格式？"第5节课会专门讲。现在只需要知道：JSON 是"给机器读的标准化格式"，我们在 Prompt 里指定了，AI 就会用这个格式输出。
  • "第2层里的 {news_list} 是什么？"这是一个占位符，表示"这里要放新闻列表，具体内容等程序运行时再填进来"。就像合同里的"__年__月__日"一样。
  • "Prompt 要多长才够？"不是越长越好。够清晰就好——四层都有、格式说清楚、异常处理写了，就足够。200字的好 Prompt 比2000字的模糊 Prompt 有用得多。
