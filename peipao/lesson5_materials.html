回顾 + 目标说明 · 0–5分钟
老师开场
上节课大家在终端看到了 AI 生成的新闻摘要——这是整个第一期的第一个真实成果。

但现在的系统只有你自己能用：要打开终端、要手动运行命令、输出也只是文字。
今天我们来做最后一步：把它变成一个真正的网页产品，任何人都能用浏览器打开，还能点按钮触发刷新。

最重要的是：今天我们继续用 Vibe Coding 的方式——你来描述你想要什么，Trae 来决定怎么实现。
需求描述练习 · 5–15分钟
为什么先写需求卡
在打开 Trae 之前，我们先花10分钟写清楚"我想要什么"。
这是 Vibe Coding 最重要的一步：需求越清晰，AI 的输出越接近你想要的。

每人发一张需求卡模板（见学员材料），独立填写5分钟，然后2人一组互相读一读， 检查对方的需求是否说得足够清楚。
老师在这10分钟关注 学员的需求描述是否具体——"好看一点"不够，要说"深色背景、卡片布局、有标题和摘要"。 如果学员写得太模糊，引导他用"感觉像某个网站"来描述。
老师 Demo · 15–40分钟
现在的系统只能在终端里跑。
怎么告诉 Trae"我想要一个可以分享给朋友看的网页产品"？
Demo 的核心教学目标
这25分钟 Demo 最重要的不是最终代码，而是向学员展示：

① 如何把需求卡里的内容翻译成 Trae 能理解的描述
② 遇到 Trae 的输出不满意时，怎么用自然语言继续调整
③ 为什么按钮触发需要一个"后端"——纯 HTML 做不到这点（让 Trae 解释，不是老师解释）

第一步：告诉 Trae 整体需求（把这段话发给 Trae AI）：

我已经有两个 Python 脚本：
- fetch_news.py：运行后从 AP News 抓取金融新闻，保存到 data/summary.json 的 raw_news 字段
- news_agent.py：运行后读取 raw_news，用 Claude AI 整理，结果保存到 items 字段

items 的格式：[{"title":"...","summary":"...","market_impact":"...","url":"..."}]

我想把这个系统做成一个网页产品，需求：

【展示】
用卡片展示 items 里的每条新闻，深色专业风格（参考金融资讯网站）
每张卡片：标题 / 摘要 / 市场影响（标签样式）/ 查看原文链接

【交互】
一个"立即刷新"按钮，点击后：
1. 触发 fetch_news.py 和 news_agent.py 的运行
2. 按钮显示 loading 状态（不能点了没反应）
3. 数据更新后自动刷新页面内容

【自动】
每天 08:00 自动运行一次

你来决定用什么技术方案，我只关心最终效果。
当 Trae 选择了 Flask，顺势解释
Trae 通常会选择用 Flask 实现——这时候老师说：

"看，Trae 选择了 Flask 来做后端。为什么需要后端？因为按钮点击要触发 Python 脚本运行， 纯 HTML 网页做不到这点——网页不能直接运行你电脑里的程序，需要一个中间人来接收请求、调用脚本、返回结果。 这个中间人就是 Flask。

我们不需要自己懂 Flask——Trae 帮我们写了，我们只需要能运行它、能验证效果。"
遇到效果不满意时，示范如何继续调整
现场演示2~3次"看效果 → 用自然语言反馈 → Trae 修改"的循环：

"卡片间距太小，文字挤在一起，能不能更宽松一点？"
"点了刷新按钮完全没反应，帮我 debug 一下"
"手机上打开，卡片横着排了，能不能变成一列？"

这个循环就是 Vibe Coding 真实的工作方式。

JSON 初识 · 88–96分钟
我们的系统能给自己看了。
但如果要让另一个 AI 来读这份摘要、给出"今天适合关注哪个板块"的建议，它能读懂吗？
引入场景
打开 data/summary.json，找到 items[0].market_impact 字段，读出一条：
比如 "受美联储政策不确定性影响，债券市场承压，利率敏感型股票面临调整压力"

问学员：如果另一个程序要从这句话里提取"影响的资产类别"和"方向是涨还是跌"——它能做到吗？

答案：很难。自然语言对人友好，对程序不友好。"承压"是什么意思？涨还是跌？程序不知道。

这就是我们第二期要解决的问题：让 Agent 的输出从"人话"变成"机器话"。
JSON 三个规则（快速讲，各2分钟）
1
每个值有名字
不能只有值，必须说明这个值是什么
"down"
→
{"direction": "down"}
2
值有类型
数字就是数字，文字就是文字，不能混
{"change": "下跌1.2%"}
→
{"change_pct": -1.2}
3
结构可以嵌套
一条记录里可以包含多个子字段，方便分类查询
{"impacts": [{"asset": "USD", "direction": "up"}, {"asset": "Gold", "direction": "down"}]}
课堂练习 把一条 market_impact 翻译成 JSON 3 分钟
1
原文："受美联储加息预期影响，美元走强，黄金承压"
2
用上面三个规则，自己设计字段，翻译成 JSON 格式（用 Claude 对话界面辅助）
3
参考答案：{"trigger": "美联储加息预期", "impacts": [{"asset": "USD", "direction": "up"}, {"asset": "Gold", "direction": "down"}]}

1
规则1：每个值有名字
不能只有值，要说明这个值是什么
"down"
{"direction": "down"}
2
规则2：值有类型
数字不加引号，文字加引号，不能混
{"change": "下跌1.2%"}

3
规则3：结构可以嵌套
一条记录里可以包含列表，列表里每项又是一个记录
