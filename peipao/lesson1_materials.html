老师开场
欢迎大家来到第一期的第一节课。在我们正式开始之前，我想先做个小调查。

我想请每位同学想一件事：你每天最花时间的"手动整理信息"工作是什么？

可以是：早上刷新闻、整理研报摘要、汇总市场数据、写晨会纪要…… 不用很复杂，就是那种"我每天都在做、但感觉挺机械的"那种。

我们来快速过一遍，每人一句话。
老师提示 在白板或共享文档上记录学员说的工作类型，按"信息来源"和"输出格式"两列整理。 这些场景后面会反复用到——学员自己说的例子比老师举的例子更有代入感。
过渡语
非常好。我注意到大家说的这些工作，基本上都有一个共同点：信息来了，要处理，然后要输出一个结论或摘要。

这节课我们要讨论的问题是：能不能把这件事交给一个 Agent 来做？
Agent 是什么？它和你现在用的 ChatGPT 有什么本质区别？我们先从这里开始。
核心概念① · 什么是 Agent · 10–30分钟
概念引入
我先问大家一个问题：你们日常用 ChatGPT 或者 Claude，是怎么用的？

通常是这样：你问一句，它答一句，你再问，它再答。你是主导的那个人，AI 是被动回应的那个。

AI Agent 不一样。Agent 更像是一个你能交代任务的实习分析师—— 你说"帮我整理今天 AP News 的金融新闻"，然后你去干别的事， 它自己去抓数据、调用工具、判断哪些重要、写出摘要，完成了再来告诉你。
普通 ChatGPT
→ 你问一句，它答一句
→ 需要你一直在场盯着
→ 不会主动用工具
→ 每次从头开始
vs
AI Agent
→ 你交代任务，它自己跑
→ 出去干活，完成了回来报告
→ 能调用工具（抓网页/查数据库）
→ 结果不对，自己重试
类比深化
想象你新来了一个实习生，你告诉他："每天早上8点，去 AP News 看看有没有和金融市场相关的新闻，整理成5条摘要，发到群里。"

他明白了之后，以后每天都会自己去做，不用你提醒，不用你盯着。做错了会自己改，实在不确定才来问你。

这就是 Agent 的核心特质：有自主性，会用工具，能处理意外情况。
常见问题预案 学员可能会问"Agent 和 RPA 有什么区别"——回答：RPA 按固定路径操作屏幕，就像录制好的宏，改一个按钮就失效。Agent 能理解语义、做判断，就像真人一样读懂上下文再决定怎么做。
核心概念② · 三种节点类型 · 30–45分钟
过渡
好，现在大家对 Agent 有了基本概念。接下来我要引入一个很重要的设计工具——节点类型。

我们在设计一个 Agent 系统的时候，不是所有步骤都需要 AI。 有些步骤适合 AI 判断，有些适合规则跑，有些必须人来拍板。 区分清楚，系统才能做得稳、做得对。
Agent 节点
需要理解语义、做判断、综合多方信息的步骤。换成固定规则做不到，或者做不好。
例：理解新闻内容、判断重要性、写摘要
Automation 节点
按固定规则执行的步骤。不需要理解，只需要执行。效率高，不出错。
例：定时触发、保存文件、格式转换、发邮件
Human 节点
必须人来决定的步骤。风险太高、责任重大、或需要主观判断。
例：最终投资决策、风控审批、策略方向判断
快问快答（互动环节）
我们来练练手。我说一个步骤，大家判断是哪种节点。快速回答就好，不需要思考太久——

① 每天早上8点自动运行一次抓取程序
答：Automation——定时执行，按规则来，不需要判断。

② 判断一条新闻对股市有没有影响
答：Agent——需要理解内容，做判断，没有固定规则。

③ 把整理好的新闻摘要保存成文件
答：Automation——固定步骤，写文件就行。

④ 决定今天是否调整持仓
答：Human——这个决定的责任和风险，不应该完全交给系统。
设计原则 Agent 节点是最贵的（调用 AI 有成本）、最慢的、最可能出现意外的。好的设计是：能用 Automation 解决的，不用 Agent；必须人来做的，不要绕过 Human 节点。
练习① · 画系统流程图 · 45–75分钟
练习说明
现在我们来做第一个练习。你们刚才说了自己每天的信息整理工作—— 现在我们聚焦在同一个场景：用 Agent 自动整理 AP News 的金融新闻，每天早上出一份摘要。

请用纸笔或者共享文档，画出这个系统的流程图。要求只有两条：
第一，每个步骤标注节点类型（Agent / Automation / Human）。
第二，必须包含至少一个失败分支——如果某一步出错了，系统怎么处理？
学员活动 画 AP News 整理 Agent 的流程图 20 分钟
1
独立完成（15分钟）：用纸笔、Docs 绘图、或飞书白板，画出你理解的流程图。
2
每个步骤旁边标注节点类型：Agent / Automation / Human
3
画出至少 1 个失败分支：比如"如果今天 AP News 无法访问，怎么办？"
4
全班分享（10分钟）：老师随机点 2~3 人展示，重点关注失败分支的设计。
点评时重点关注 （1）有没有把 Agent 节点用在不需要 AI 的步骤上（常见：把"保存文件"设计成 Agent）
（2）失败分支是否具体（"出错就报错"不够，要说明报什么错、然后呢）
（3）有没有遗漏关键步骤（常见：没有"数据存在哪里"的步骤）
学员常见设计问题 问题：把所有步骤都设计为 Agent 节点
引导：问他"这一步需要理解什么？还是只是按规则执行？"

问题：流程图只有顺利路径，没有失败分支
引导：问他"如果今天 AP News 的页面改版了，系统还能抓到新闻吗？"
练习② · 填写角色卡 · 75–90分钟
引入角色卡
流程图告诉我们系统长什么样。但 Agent 节点里面是什么？它怎么知道自己该做什么？

在我们这门课里，我们用"角色卡"来定义每一个 Agent 节点。 角色卡就是这个 Agent 的岗位说明书：它是谁、看什么、做什么、交出什么、做不到时怎么办。

这节课我们填第一版——全部用自然语言，不需要任何代码。 下节课我们会把角色卡升级成真正的 Prompt。
学员活动 填写 News Agent 角色卡 15 分钟
1
打开学员材料页面（切换到"学员材料"视图），找到角色卡模板
2
填写 5 个字段：目标 / 输入 / 处理 / 输出 / 失败时
3
重点：失败时字段不能留空，必须有具体说明
老师示范答案（供参考，不发给学员） 目标：每天从 AP News 中找出金融市场最需要关注的信息
输入：今日 AP News 新闻列表，最多20条，每条含标题+时间+摘要
处理：筛选金融相关新闻，按重要性排序，为每条提炼标题、摘要和市场关联
输出：最多5条新闻，每条包含原标题 / 2~3句摘要 / 为什么金融从业者需要关注
失败时：若今日无相关新闻，输出"今日无重大金融新闻"，不编造内容
常见问题与参考回答
  • "Agent 学起来难吗？" 我们整个课程都不要求你写代码。你只需要能把需求说清楚——就像给实习生写任务说明。难的不是技术，是把你的需求想清楚。鼓励
  • "我们做的这个系统是真的能用吗？" 是的。第4节课结束时，你会有一个真实运行的 Python 程序；第5节课结束，你会有一个有真实网址的网页。不是 demo，是可以每天真正跑的系统。期待感
  • "AP News 为什么选这个？" 三个原因：完全免费、内容权威、页面结构相对稳定，适合课程。你自己感兴趣的数据源，后面可以替换。
  • "Automation 和 Agent 有时候边界很模糊怎么办？" 好问题。判断标准就一条：这一步需要理解语义或做主观判断吗？需要 → Agent；不需要 → Automation。模糊的时候先选 Automation，系统更稳定。
预告 · 90–100分钟
下节课预告
今天我们完成了系统的第一步设计：你知道了系统该长什么样，也给 News Agent 写了角色卡。

但现在有一个问题：AI 怎么知道它要按角色卡工作？ 你总不能真的把角色卡贴给它看，然后期待它自动理解。

下节课我们解决这个问题——Prompt Engineering。 我们会把今天写的角色卡，变成一份 AI 真的能读懂、真的会按照它执行的指令。

下节课之前不需要任何准备，带上今天填好的角色卡就行。

口述验收——能说出：Agent 做什么、不做什么、失败时怎么处理、是哪种节点
