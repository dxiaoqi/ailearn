Agent 的本质是：把「每一步做什么」这个决策交给 LLM 来判断。它不是执行固定脚本，而是每轮都重新看一遍现有信息，然后决定下一步。

这带来了巨大的灵活性——也带来了新的风险：你不再能预测它会走哪条路，也不知道它什么时候会「停下来」。
◈
三个你必须理解的概念
概念 1
Context Window · 记忆容量
LLM 每次只能「看到」一个固定大小的文本窗口。每轮循环，它看到的内容都在增长——上一轮的观测、自己之前的回复、所有历史记录都要塞进这个窗口。

窗口塞满了就会截断——就像你的工作台太小，只能把最早的文件推到地上
概念 2
Agent Loop · 循环机制
Agent 不是一问一答，而是不断循环：观测环境 → 推理下一步 → 执行行动 → 再次观测。这个循环会一直跑，直到满足终止条件。

「执行轮次」越多，完成的任务越复杂，但消耗的 Token 也越多
概念 3
终止条件 · Agent 的刹车
Agent 不会自动知道任务完成了。你必须告诉它「完成的样子是什么」，或者设置一个「最多跑多少轮」的安全网。

缺少终止条件 = 无限循环 = 无限账单

⊡
Loop 靠终止条件停下
三道保障——缺一就有风险：
✓ LLM 主动输出终止信号（如 TASK_DONE）
✓ 达到 max_turns 上限强制退出
✓ 外部中断（用户取消 / 超时）
